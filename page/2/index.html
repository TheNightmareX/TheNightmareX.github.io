HTML中的`script`标签

发表于 2020-10-25 更新于 2021-07-16

传统 script

传统 script 指所有非 ES6 模块化结构的 script

<body>中的<script>

阻塞 dom 加载，下载并执行完毕后继续 dom 的加载
这是由于这类 script 可能会用于执行渲染操作以改变向用户呈现出的第一眼效果
但如果该 script 的内容是其他操作
则会影响页面呈现速度，降低用户体验，且语义性差，不符合人类直觉
如果 script 不改变第一眼呈现时的效果，则不建议使用这种
通常只有 UI 框架的 script 会放在<body>的底部，因为它会改变第一眼的呈现效果

<head>中的<script>+window.onload

下载过程阻塞 dom 加载
但 window.onload 监听器中的主要代码会等待 dom 加载渲染完毕后执行
同样会影响页面呈现速度，语义性差，但较上一种要好些
不论你的 script 是做什么的，都不推荐这种做法，因为已经有替代方案

<head>中的<script defer>

script 标签放在<head>中并附上 defer 属性
则该 script 的下载过程与 dom 加载同时进行，并会等待 dom 加载完毕后执行
对于不改变第一呈现效果但又要等待 dom 加载完毕后执行的 script，此方案是首选

<head>中的<script async>

<head>中的 script 加上 async 属性
则该 script 的下载过程与 dom 加载同时进行，下载完毕后立刻执行
执行过程会阻塞 dom 加载
适用于不操作 dom 的 script（有这种脚本？）
用的很少

ES6 模块化 script

指通过 import 和 export 语句的结构化 script
在 head 中只有一个<script type=”module”>引入顶级模块（入口模块）
其他子模块都由该顶级模块 import 进来
所有模块的下载都是与 dom 加载同时进行的，并在 dom 加载完毕后执行顶级模块

理解Web开发中的前后端

发表于 2020-10-24 更新于 2021-07-01

浏览一个 Web 页面：浏览器->服务器->数据库->服务器->浏览器
其中需要编程的有浏览器上的 HTML、CSS、JS 和服务器上各类语言开发
于是我曾这么认为：js 之类的便是前端，django 一类的便是后端
直到我学了 Vue……

事实上
负责视图层的都叫前端，负责数据库操作的都叫后端

前端可以是几个文件，也可以是大规模项目，甚至可以不涉及 JS
例如 Vue 生态系统中的 Vue CLI 便是用来开发大规模前端的
大前端便是在专用目录中使用大型框架开发完 HTML、CSS、JS 后，使用各种构建工具打包（多个文件整合成一个文件）为最终在浏览器上运行的文件
而 Django 中的模板系统也是一种前端技术
只要涉及 HTML、CSS、JS 的都叫前端

后端是开发数据库 API 的
它不返回 HTML 页面，而是返回 JSON 数据，用以供 Web 前端或是桌面 APP 调用

Django 是一个前后端不分离的 Web 框架。它的前端模板系统和后端数据库系统都是它的亮点，但前端毕竟是 js 的专场，在很多项目中，django 都扮演的是后端的角色。django rest framework 是比较热门的基于 django 的用来开发 RESTful API 的框架。

CentOS7宝塔面板部署Django-SQLite项目

发表于 2020-10-12 更新于 2021-07-01

服务端应用程序部署

  • 升级 SQLite3
    Django3.1 需要 3.8.3 以上版本的 SQLite
    系统自带的 SQLite3 则是 3.7
    不升级无法启动项目
    从SQLite 官网下载最新的 SQLite3 源码包并解包
    编译安装，设置 prefix 以覆盖系统自带的 SQLite3
    若不设置 prefix，则默认安装到_/_usr/local
    而动态链接库搜索路径中，/usr 先于/usr/local
    因此 Python 依旧会调用旧的 SQLite3
    除非将旧的删干净或者手动设置动态链接库搜索路径（不推荐）
  • 软件商店：安装Python 项目管理器1.9
  • 在 Python 项目管理器 - 版本管理中安装 Python
    安装Python 项目管理器时会自动安装 pyenv 以管理 Python 版本
    若安装 Python 失败，请尝试使用 pyenv 自行本地安装
  • 在Python 项目管理器中添加项目
    以下是添加项目时一些容易引起歧义的设置项：
    • 路径：Django 项目根目录（manage.py 所在目录）
    • 端口：随便选一个服务器未被占用的端口号（不是最终端口号）
    • 安装模块依赖：会自动创建虚拟环境“项目名称_venv”并安装 requirements.txt 中的第三方库
  • Monkey Patch on Django
    可能是 Django3.1 的 bug?
    如果项目适用 sqlite
    则在操作数据库时依旧会抛出异常：需要 3.8.3 以上的 SQLite
    只能停止项目，Monkey Patch 一下 Django 的代码临时修复一下
    xxx_venv/lib/python3.x/site-packages/django/db/backends/sqlite3/base.py
    • 复制 214 行到 215 行，并减少一层缩进
    • 注释掉 208-214 行
  • 进入项目目录部署 Django 项目

后端完成部署，接下来是把项目映射到 Nginx

服务器部署

  • 在Python 项目管理器中，映射你的项目
    • 单击映射
    • 输入要绑定的域名
      这个域名不能带有端口号，要指定端口号，请在网站中为该网站随意增加一个绑定域名，删除原先的错误端口域名，新增正确端口域名，删除随意增加的那个域名
      该域名将用来
      • 作为 Nginx 网站名
      • 在/www/wwwroot/中以该域名作为目录名创建网站根目录
      • 设置该 Nginx 映射网站的绑定域名为该域名的 80 端口
        当服务器 80 端口收到请求
        服务器会判断请求的域名绑定到了哪个网站上
        随后将数据包转发给那个网站
  • 在 Nginx 配置文件中配置静态文件路径
    Django 项目为服务端应用程序，在生产环境中，分发静态文件应当直接由服务器完成，不应交由 Django 完成
