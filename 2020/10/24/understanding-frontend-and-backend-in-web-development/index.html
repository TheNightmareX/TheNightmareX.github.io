理解Web开发中的前后端

发表于 2020-10-24 更新于 2021-07-21

浏览一个 Web 页面：浏览器->服务器->数据库->服务器->浏览器
其中需要编程的有浏览器上的 HTML、CSS、JS 和服务器上各类语言开发
于是我曾这么认为：js 之类的便是前端，django 一类的便是后端
直到我学了 Vue……

事实上
负责视图层的都叫前端，负责数据库操作的都叫后端

前端可以是几个文件，也可以是大规模项目，甚至可以不涉及 JS
例如 Vue 生态系统中的 Vue CLI 便是用来开发大规模前端的
大前端便是在专用目录中使用大型框架开发完 HTML、CSS、JS 后，使用各种构建工具打包（多个文件整合成一个文件）为最终在浏览器上运行的文件
而 Django 中的模板系统也是一种前端技术
只要涉及 HTML、CSS、JS 的都叫前端

后端是开发数据库 API 的
它不返回 HTML 页面，而是返回 JSON 数据，用以供 Web 前端或是桌面 APP 调用

Django 是一个前后端不分离的 Web 框架。它的前端模板系统和后端数据库系统都是它的亮点，但前端毕竟是 js 的专场，在很多项目中，django 都扮演的是后端的角色。django rest framework 是比较热门的基于 django 的用来开发 RESTful API 的框架。
