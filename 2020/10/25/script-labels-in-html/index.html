HTML中的`script`标签

发表于 2020-10-25 更新于 2021-07-16

传统 script

传统 script 指所有非 ES6 模块化结构的 script

<body>中的<script>

阻塞 dom 加载，下载并执行完毕后继续 dom 的加载
这是由于这类 script 可能会用于执行渲染操作以改变向用户呈现出的第一眼效果
但如果该 script 的内容是其他操作
则会影响页面呈现速度，降低用户体验，且语义性差，不符合人类直觉
如果 script 不改变第一眼呈现时的效果，则不建议使用这种
通常只有 UI 框架的 script 会放在<body>的底部，因为它会改变第一眼的呈现效果

<head>中的<script>+window.onload

下载过程阻塞 dom 加载
但 window.onload 监听器中的主要代码会等待 dom 加载渲染完毕后执行
同样会影响页面呈现速度，语义性差，但较上一种要好些
不论你的 script 是做什么的，都不推荐这种做法，因为已经有替代方案

<head>中的<script defer>

script 标签放在<head>中并附上 defer 属性
则该 script 的下载过程与 dom 加载同时进行，并会等待 dom 加载完毕后执行
对于不改变第一呈现效果但又要等待 dom 加载完毕后执行的 script，此方案是首选

<head>中的<script async>

<head>中的 script 加上 async 属性
则该 script 的下载过程与 dom 加载同时进行，下载完毕后立刻执行
执行过程会阻塞 dom 加载
适用于不操作 dom 的 script（有这种脚本？）
用的很少

ES6 模块化 script

指通过 import 和 export 语句的结构化 script
在 head 中只有一个<script type=”module”>引入顶级模块（入口模块）
其他子模块都由该顶级模块 import 进来
所有模块的下载都是与 dom 加载同时进行的，并在 dom 加载完毕后执行顶级模块
